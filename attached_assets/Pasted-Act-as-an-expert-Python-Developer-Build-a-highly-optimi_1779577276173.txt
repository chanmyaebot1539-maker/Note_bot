{
  "_id": ObjectId,
  "creator_id": 12345678,        // Telegram User ID (or OWNER_ID)
  "creator_name": "John Doe",    // Full Name or Username of the creator
  "command_name": "love",        // Stored without leading slash

    {"type": "text", "content": "Believe in yourself..."},
    {"type": "photo", "content": "file_id_here", "caption": "Optional"}
  ]
}

### FUNCTIONAL ARCHITECTURE & FLOW

1. PUBLIC & OWNER COMMANDS:
- Commands created by the OWNER_ID are "Global" and can be viewed/triggered via Keyboard/Inline Buttons by all users.
- Commands created by regular users are "Private" and only visible/accessible to that specific creator_id.

2. MAIN MENU LAYOUT:
- Show a persistent custom keyboard markup.
- For all users: Display "Create Your Own Command" button alongside active Global Commands.
- If a regular user has custom commands, display them inside a separate dynamically loaded inline section/keyboard.

3. COMMAND CREATION WORKFLOW (State Machine / ConversationHandler):
- Triggered by clicking "Create Your Own Command" or typing `/createcommand`.
- Step 1: Ask for Command Name (Validate: only Latin letters, numbers, and underscores).
- Step 2: Ask for Reply Content (Support Text, Photos, Videos, Documents). Allow adding multiple messages via an "Add Messages to Command" routine or click "Save" to finish.
- Save the session to MongoDB under the user's explicit `creator_id` and `creator_name`.

4. BOT OWNER CONTROL PANEL:
- Accessible ONLY if user_id == OWNER_ID.
- Show an "Admin Panel" button in the menu. Inside, display a list of Users who have created commands, formatted as buttons: `[User Name/Username]`
- Clicking a user's name displays all custom commands created by that specific user.
- Provide a "Delete Command" feature under each listed user command so the Owner can moderate and remove inappropriate content instantly.

5. WEB SERVER FOR RENDER (Always On):
- Integrate a simple Quart or Flask background web server running on port 8080.
- Implement a `GET /` health-check route that replies "OK" to allow UptimeRobot to ping it successfully and prevent Render from sleeping.
